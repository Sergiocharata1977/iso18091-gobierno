# Guia de activacion: de cero a tu SGC en marcha

Si estas empezando con Don Candido / 9001app, esta guia te muestra el recorrido completo para dejar tu organizacion activa y lista para trabajar en ISO 9001.

El proceso esta pensado en 3 etapas consecutivas. Primero se crea la base del sistema, despues se completa la estrategia de gestion y finalmente se empieza a ejecutar el plan de trabajo desde el roadmap.

## Vista general del camino

1. **Wizard de provision** (`/onboarding`): cargas datos clave y el sistema arma la base inicial.
2. **Estrategia** (`/planificacion-revision-direccion?onboarding=1`): definis identidad, alcance y lineamientos.
3. **Roadmap** (`/mi-sgc/roadmap`): avanzas por fases ISO 9001 con tareas concretas.

Si completas estas 3 etapas en orden, quedas con una estructura inicial lista para trabajar de forma sostenida.

## Etapa 1: Wizard de provision (/onboarding)

### Que hace el usuario

En esta primera pantalla avanzas por 5 pasos:

1. Seleccion de sistema.
2. Datos de la empresa.
3. Norma a aplicar.
4. Procesos ISO.
5. Confirmacion final.

Tu objetivo aca es cargar informacion base con criterio practico. No hace falta que todo quede perfecto en esta instancia: despues vas a poder ajustar.

### Que hace el sistema

Cuando confirmas, Don Candido crea automaticamente la estructura inicial de tu organizacion:

- Procesos base alineados a ISO 9001.
- Puntos de norma asociados para empezar a trabajar.

Al terminar, aparece el boton **"Continuar: completar estrategia ->"** para pasar a la segunda etapa.

### Cuanto tarda aproximadamente

- Carga inicial: **10 a 20 minutos**.
- Puede ser menos si ya tenes claros los datos de empresa y el enfoque de procesos.

## Etapa 2: Estrategia (/planificacion-revision-direccion?onboarding=1)

### Cuando se habilita

Esta etapa se abre automaticamente cuando:

- La provision inicial fue completada exitosamente.
- Tu usuario es el **owner** o responsable de la organizacion.

Si no sos el owner, podes cargar informacion pero el cierre final depende de quien tenga ese rol.

### Que hace el usuario

Tenes que completar 5 secciones:

1. **Identidad**.
2. **Alcance**.
3. **Contexto**.
4. **Estructura**.
5. **Politicas**.

Cada seccion tiene campos concretos que el sistema valida en segundo plano. En total, hay **12 checks** de completitud.

El foco no es escribir largo, sino dejar definiciones utiles para operar:

- Quienes son.
- Que cubre el sistema.
- En que contexto trabajan.
- Como esta organizada la empresa.
- Que lineamientos de politica van a seguir.

### Que hace el sistema

A medida que completas campos, Don Candido va marcando checks internos. Cuando los 12 checks estan completos, se habilita el boton:

**"Generar procesos base y modelos (borrador)"**.

Al hacer clic:

- El sistema genera borradores iniciales.
- Luego redirige automaticamente a **`/mi-sgc/roadmap`**.

### Cuanto tarda aproximadamente

- Completado de estrategia: **20 a 40 minutos**.
- Generacion de borradores: **menos de 1 minuto**.

## Etapa 3: Roadmap (/mi-sgc/roadmap)

### Que hace el usuario

Desde aca empezas la ejecucion del sistema por fases ISO 9001:

1. Diagnostico.
2. Planificacion.
3. Diseno.
4. Implementacion.
5. Verificacion.
6. Mejora.

El avance es manual por tarea. Tu equipo marca progreso a medida que va haciendo trabajo real.

### Que hace el sistema

Don Candido te muestra el camino completo en fases y te permite administrar el avance de forma ordenada.

Ademas, el roadmap queda siempre accesible desde el sidebar de **Mi SGC**, para volver cuando quieras sin depender del flujo inicial.

### Cuanto tarda aproximadamente

- Primer ingreso y orden inicial: **15 a 30 minutos**.
- Ejecucion completa: depende del tamano de la empresa y madurez del sistema.

## Preguntas frecuentes

### Que pasa si no soy el owner?

Podes cargar parte de la informacion, pero la etapa de estrategia y su cierre dependen del usuario owner. Si sos gerente o responsable de calidad sin rol owner, pedile al owner que:

1. Ingrese y complete la parte final.
2. O te asigne el rol correspondiente para continuar.

### Puedo volver a generar procesos base y modelos?

La generacion se habilita cuando la estrategia cumple los 12 checks. En uso normal, se hace para crear la base inicial. Si necesitas regenerar por cambios grandes de enfoque, primero revisa impacto en lo que ya fue editado por el equipo para no pisar trabajo previo.

### Los borradores se publican solos?

No. Se generan como **borrador** para que el equipo los revise, ajuste y valide antes de tratarlos como definitivos. Esto evita automatismos que puedan dejar documentos o procesos "cerrados" sin revision humana.

## Cierre

El objetivo de este onboarding no es "llenar pantallas", sino dejar una base clara para gestionar calidad en serio. Si completas las 3 etapas con foco practico, en poco tiempo podes dejar preparada la estructura inicial de tu SGC y empezar a ejecutar desde el roadmap.
